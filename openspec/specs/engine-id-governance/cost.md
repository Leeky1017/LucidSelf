# 引擎ID治理 — 成本目标与预算阈值（规格层）

本文件定义 `engine-id-governance` 的成本目标、预算阈值与告警策略。禁止写实现细节；仅定义 MUST/SHALL 与可判定口径。

## 1) 成本维度（最小）

- **存储**：注册表快照、生命周期事件、弃用/替代关系、审计记录
- **计算**：治理写路径校验、引用解析与一致性检查
- **可观测**：日志与审计导出带来的 IO/存储成本

## 2) 预算阈值（最小）

以下阈值用于早期商业化基线（Gate-0）；后续可按真实负载修订，但口径必须可执行、可判定：

- 审计记录写入量：日均 `ls_engine_governance_audit_records_total` SHOULD ≤ 1,000,000
- 生命周期事件写入量：日均事件数 SHOULD ≤ 100,000
- 审计导出体量：单次导出 SHOULD ≤ 1GB（超过则必须分片/分页导出；策略实现可选）

## 3) 告警策略（最小）

- 超过预算阈值（连续 3 个评估窗口）MUST 告警，并形成可审计事件。
- 告警 MUST 可关联到：时间窗口、影响面（读/写/导出）、关键标识符（若适用，至少 `version_id`）。
