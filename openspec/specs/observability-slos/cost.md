# 可观测性与SLO — 成本目标与预算阈值（规格层）

本文件定义 `observability-slos` 的成本目标、预算阈值与告警策略。禁止写实现细节；仅定义 MUST/SHALL 与可判定口径。

## 1) 成本维度（最小）

- **日志**：写入量、存储量、索引成本
- **追踪**：采样率、span 数量、存储量
- **指标**：时间序列数量（标签基数）、存储与查询成本

## 2) 预算阈值（最小）

以下阈值用于早期商业化基线（Gate-0）；后续可按真实负载修订，但口径必须可执行、可判定：

- 指标时间序列：时间序列数量异常抬升 MUST 告警（通常由高基数标签导致）
- 日志写入量：日均日志量 SHOULD 在预算内（预算单位由运行方定义，但必须可审计）
- trace 采样率：采样率升高导致成本超预算 MUST 告警并可审计

## 3) 告警策略（最小）

- 超过预算阈值（连续 3 个评估窗口）MUST 告警，并形成可审计事件。
- 告警 MUST 可关联到：时间窗口、影响面（logs/traces/metrics）、触发原因（例如 `HIGH_CARDINALITY_LABEL`）。
