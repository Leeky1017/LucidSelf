# 语料治理 — 安全与隔离边界（规格层）

本文件定义 `corpus-governance` 的安全与隔离最小契约。禁止写实现细节；仅定义 MUST/SHALL 与可判定口径。

## 1) 访问控制（最小）

- 资产登记、许可约束、血缘与退役属于治理级能力：
  - 写操作 MUST 受高权限控制；未授权 MUST 被拒绝并审计（`UNAUTHORIZED`）。
- 资产元数据对外暴露 SHOULD 默认最小化；许可条款与受限字段 MUST 受权限控制。

## 2) 隔离边界（user_id/org_id）

- 若资产访问按租户隔离（例如私有语料），系统 MUST 遵循 `user_id/org_id` 隔离边界；跨边界 MUST 被拒绝并审计。
- 若资产为全局域，系统 MUST 明确其为全局域，并通过许可/访问控制限制使用范围。
