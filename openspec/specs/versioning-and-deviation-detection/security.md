# 版本化与偏差检测 — 安全与隔离边界（规格层）

本文件定义 `versioning-and-deviation-detection` 的安全与隔离最小契约。禁止写实现细节；仅定义 MUST/SHALL 与可判定口径。

## 1) 访问控制（最小）

- 写操作（新增版本清单、发布兼容性声明、生成阻断记录）属于治理级能力：
  - 未授权请求 MUST 被拒绝（`UNAUTHORIZED`）并写入审计记录。
- 读操作（查询版本/兼容性/偏差报告）SHOULD 默认最小化暴露，避免泄露内部实现细节；超出范围 MUST 受权限控制。

## 2) 隔离边界（user_id/org_id）

- 若偏差检测或报告与用户数据相关（例如用户可见输出的偏差），系统 MUST 绑定并遵循 `user_id/org_id` 隔离边界；跨边界 MUST 被拒绝并审计。
- 若为全局治理域（与单一用户无关），系统 MUST 明确其为全局域并仅允许被授权主体调用。
