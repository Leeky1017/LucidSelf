# 版本化与偏差检测 — 成本目标与预算阈值（规格层）

本文件定义 `versioning-and-deviation-detection` 的成本目标、预算阈值与告警策略。禁止写实现细节；仅定义 MUST/SHALL 与可判定口径。

## 1) 成本维度（最小）

- **存储**：版本清单、兼容性声明、偏差报告、阻断记录
- **计算**：兼容性判定与偏差检测

## 2) 预算阈值（最小）

以下阈值用于早期商业化基线（Gate-0）；后续可按真实负载修订，但口径必须可执行、可判定：

- 偏差报告生成量：异常抬升 MUST 告警（可能意味着回归大面积失败）
- 阻断记录生成量：异常抬升 MUST 告警（可能意味着门禁口径或发布流程问题）

## 3) 告警策略（最小）

- 超过预算阈值（连续 3 个评估窗口）MUST 告警，并形成可审计事件。
- 告警 MUST 可关联到：时间窗口、`gate`、`deviation_type`（若适用）。
