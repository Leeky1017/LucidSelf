# 启动性与回归基线 — 安全与隔离边界（规格层）

本文件定义 `bootability-and-regression-baseline` 的安全与隔离最小契约。禁止写实现细节；仅定义 MUST/SHALL 与可判定口径。

## 1) 访问控制（最小）

- 回归基线定义与门禁阻断记录属于治理级产物：
  - 变更（新增/修改/删除基线、调整阈值）MUST 受高权限控制；未授权 MUST 被拒绝并审计。
- 回归结果与阻断记录的访问 SHOULD 默认最小化；超出范围 MUST 受权限控制。

## 2) 隔离边界（user_id/org_id）

- 若回归结果或阻断记录包含用户作用域信息，系统 MUST 遵循 `user_id/org_id` 隔离边界；跨边界 MUST 被拒绝并审计。
- 若为全局治理域，系统 MUST 明确其为全局域并仅允许被授权主体调用。
