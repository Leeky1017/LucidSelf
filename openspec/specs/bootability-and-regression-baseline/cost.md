# 启动性与回归基线 — 成本目标与预算阈值（规格层）

本文件定义 `bootability-and-regression-baseline` 的成本目标、预算阈值与告警策略。禁止写实现细节；仅定义 MUST/SHALL 与可判定口径。

## 1) 成本维度（最小）

- **计算**：回归运行耗时与资源消耗
- **存储**：回归结果、阻断记录与版本关联报告

## 2) 预算阈值（最小）

- 回归运行量/时长异常抬升 MUST 告警（可能意味着回归失控或发布节奏异常）。
- 阻断记录异常抬升 MUST 告警（可能意味着大面积回归失败）。

## 3) 告警策略（最小）

- 超过预算阈值（连续 3 个评估窗口）MUST 告警，并形成可审计事件。
- 告警 MUST 可关联到：时间窗口、`suite_id`（若适用）、`version_id`（以日志/审计方式，不作为默认 metric 标签）。
