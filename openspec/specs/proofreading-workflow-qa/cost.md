# 校对工作流与QA — 成本目标与预算阈值（规格层）

本文件定义 `proofreading-workflow-qa` 的成本目标、预算阈值与告警策略。禁止写实现细节；仅定义 MUST/SHALL 与可判定口径。

## 1) 成本维度（最小）

- **存储**：校对任务、缺陷记录、批准记录、QA 报表、审计日志与导出包
- **计算**：指标聚合、证据链校验与门禁检查
- **可观测**：结构化日志与审计导出带来的 IO/存储成本

## 2) 预算阈值（最小）

以下阈值用于 `资产门禁` 的最小商业化基线；后续可按真实负载修订，但口径必须可执行、可判定：

- 单次审计导出体量：单次导出 SHOULD ≤ 1GB（超过则必须分片/分页导出；策略实现可选）
- 单条缺陷记录体量：序列化后的 `DefectRecord` SHOULD ≤ 64KB（超过则必须拒绝或降级）
- 单条 QA 报表体量：序列化后的 `QAMetricsReport` SHOULD ≤ 512KB（超过则必须拒绝或降级）

## 3) 告警策略（最小）

- 超过预算阈值（连续 3 个评估窗口）MUST 告警，并形成可审计事件。
- 告警 MUST 可关联到：时间窗口、影响面（存储/计算/导出）、拒绝原因/阻断类型（若适用）。
