# 生产默认值（禁止Mock）— 成本目标与预算阈值（规格层）

本文件定义 `production-defaults-no-mock` 的成本目标、预算阈值与告警策略。禁止写实现细节；仅定义 MUST/SHALL 与可判定口径。

## 1) 成本维度（最小）

- **审计**：违规审计与失败原因记录的写入量与导出体量
- **计算**：配置校验与违规检测

## 2) 预算阈值（最小）

以下阈值用于早期商业化基线（Gate-0）；后续可按真实负载修订，但口径必须可执行、可判定：

- 启动失败与违规审计量异常抬升 MUST 告警（可能意味着配置/环境漂移或违规引入）

## 3) 告警策略（最小）

- 超过预算阈值（连续 3 个评估窗口）MUST 告警，并形成可审计事件。
- 告警 MUST 可关联到：时间窗口、`environment`、`reason_code`（若适用）。
