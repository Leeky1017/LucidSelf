# 生产默认值（禁止Mock）— 安全与隔离边界（规格层）

本文件定义 `production-defaults-no-mock` 的安全与隔离最小契约。禁止写实现细节；仅定义 MUST/SHALL 与可判定口径。

## 1) 访问控制（最小）

- 生产默认值规则与环境隔离约束属于治理级配置：
  - 变更（新增/修改/删除规则）MUST 受高权限控制；未授权 MUST 被拒绝并审计（`UNAUTHORIZED`）。
- 启动失败原因与违规审计记录的访问 SHOULD 默认最小化；超出范围 MUST 受权限控制。

## 2) 隔离边界（user_id/org_id）

- 若配置或审计与特定 `user_id/org_id` 相关（例如租户级配置），系统 MUST 遵循隔离边界；跨边界 MUST 被拒绝并审计。
- 若为全局治理域（与单一用户无关），系统 MUST 明确其为全局域并仅允许被授权主体调用。
