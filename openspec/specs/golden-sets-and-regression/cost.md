# Golden Set与回归评测 — 成本目标与预算阈值（规格层）

本文件定义 `golden-sets-and-regression` 的成本目标、预算阈值与告警策略。禁止写实现细节；仅定义 MUST/SHALL 与可判定口径。

## 1) 成本维度（最小）

- **存储**：golden set 清单、覆盖率报告、回归阈值与结果、漂移告警、审计记录与导出包
- **计算**：覆盖率统计、回归运行与阈值判定、漂移检测
- **可观测**：日志与审计导出带来的 IO/存储成本

## 2) 预算阈值（最小）

以下阈值用于早期商业化基线（Gate-1）；后续可按真实负载修订，但口径必须可执行、可判定：

- 回归运行量：日均 `ls_golden_sets_regression_runs_total` SHOULD ≤ 200,000
- 漂移告警量：日均 `ls_golden_sets_drift_alerts_total` SHOULD ≤ 50,000
- 审计导出体量：单次导出 SHOULD ≤ 1GB（超过则必须分片/分页导出；策略实现可选）

## 3) 告警策略（最小）

- 超过预算阈值（连续 3 个评估窗口）MUST 告警，并形成可审计事件。
- 告警 MUST 可关联到：时间窗口、影响面（回归/漂移/导出）、失败/阻断类型（若适用）。
