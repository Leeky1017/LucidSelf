# 计费与权益 — 成本目标与预算阈值（规格层）

本文件定义 `billing-and-entitlements` 的成本目标、预算阈值与告警策略。禁止写实现细节；仅定义 MUST/SHALL 与可判定口径。

## 1) 成本维度（最小）

- **存储**：用量事件、计费记录、争议证据链、审计记录与导出包
- **计算**：计量汇总、计费计算、漂移检测与证据链校验
- **可观测**：结构化日志与审计导出带来的 IO/存储成本

## 2) 预算阈值（最小）

以下阈值用于早期商业化基线（Gate-3）；后续可按真实负载修订，但口径必须可执行、可判定：

- 用量事件入账量：日均 `ls_billing_usage_events_total` SHOULD ≤ 50,000,000
- 计费记录生成量：日均 `ls_billing_billing_records_total` SHOULD ≤ 5,000,000
- 单次审计导出体量：单次导出 SHOULD ≤ 1GB（超过则必须分片/分页导出；策略实现可选）
- 单条计费记录体量：序列化后的 `BillingRecord`（含 line_items）SHOULD ≤ 512KB（超过则必须拒绝或降级）

## 3) 告警策略（最小）

- 超过预算阈值（连续 3 个评估窗口）MUST 告警，并形成可审计事件。
- 告警 MUST 可关联到：时间窗口、影响面（入账/计费/导出）、拒绝原因/阻断类型（若适用）。
