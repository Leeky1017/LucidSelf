# LLM叙述层 — 成本目标与预算阈值（规格层）

本文件定义 `llm-narrative-layer` 的成本目标、预算阈值与告警策略。禁止写实现细节；仅定义 MUST/SHALL 与可判定口径。

## 1) 成本维度（最小）

- **LLM tokens**：输入/输出 tokens 及重试带来的增量
- **存储**：叙述输出记录、引用绑定、内容安全审计、审计日志与导出包
- **计算**：引用解析校验、偏差检测与门禁检查
- **可观测**：结构化日志与审计导出带来的 IO/存储成本

## 2) 预算阈值（最小）

以下阈值用于 Gate-2 的最小商业化基线；后续可按真实负载修订，但口径必须可执行、可判定：

- 单次叙述请求总 tokens：p95 SHOULD ≤ 6,000（超过则必须告警或阻断进入扩量阶段）
- 单次叙述输出体量：序列化后的 `NarrativeOutput` SHOULD ≤ 256KB（超过则必须拒绝或降级）
- 审计导出体量：单次导出 SHOULD ≤ 1GB（超过则必须分片/分页导出；策略实现可选）

## 3) 告警策略（最小）

- 超过预算阈值（连续 3 个评估窗口）MUST 告警，并形成可审计事件。
- 告警 MUST 可关联到：时间窗口、影响面（tokens/存储/导出）、拒绝原因/阻断类型（若适用）。
