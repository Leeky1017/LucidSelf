# 融合引擎可解释性 — 成本目标与预算阈值（规格层）

本文件定义 `fusion-engine-explainability` 的成本目标、预算阈值与告警策略。禁止写实现细节；仅定义 MUST/SHALL 与可判定口径。

## 1) 成本维度（最小）

- **存储**：解释记录、回放输入集、证据对齐引用、审计日志与导出包
- **计算**：证据解析校验、偏差检测、门禁检查与回放校验
- **可观测**：结构化日志与审计导出带来的 IO/存储成本
- **LLM tokens（若适用）**：若解释生成依赖 LLM，则包含输入/输出 tokens 及重试增量

## 2) 预算阈值（最小）

以下阈值用于 Gate-2 的最小商业化基线；后续可按真实负载修订，但口径必须可执行、可判定：

- 单条解释记录体量：序列化后的 `ExplanationRecord` SHOULD ≤ 128KB（超过则必须拒绝或降级）
- 单次回放输入集体量：序列化后的 `PlaybackInputSet` SHOULD ≤ 512KB（超过则必须拒绝或降级）
- 单次审计导出体量：单次导出 SHOULD ≤ 1GB（超过则必须分片/分页导出；策略实现可选）

## 3) 告警策略（最小）

- 超过预算阈值（连续 3 个评估窗口）MUST 告警，并形成可审计事件。
- 告警 MUST 可关联到：时间窗口、影响面（存储/计算/导出/LLM tokens）、拒绝原因/阻断类型（若适用）。
