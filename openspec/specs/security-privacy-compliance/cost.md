# 安全/隐私/合规 — 成本目标与预算阈值（规格层）

本文件定义 `security-privacy-compliance` 的成本目标、预算阈值与告警策略。禁止写实现细节；仅定义 MUST/SHALL 与可判定口径。

## 1) 成本维度（最小）

- **审计**：审计记录写入量与导出体量
- **隐私请求**：导出/删除作业的计算与存储成本
- **事件响应**：复盘与证据留存成本

## 2) 预算阈值（最小）

以下阈值用于早期商业化基线（Gate-0）；后续可按真实负载修订，但口径必须可执行、可判定：

- 审计记录写入量：日均 `ls_security_audit_records_total` SHOULD ≤ 5,000,000
- 隐私请求导出体量：单次导出 SHOULD ≤ 5GB（超过则必须分片/分页导出；策略实现可选）

## 3) 告警策略（最小）

- 超过预算阈值（连续 3 个评估窗口）MUST 告警，并形成可审计事件。
- 告警 MUST 可关联到：时间窗口、影响面（审计/导出/删除）、触发原因。
