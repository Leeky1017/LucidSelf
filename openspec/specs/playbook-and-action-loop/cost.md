# Playbook与行动闭环 — 成本目标与预算阈值（规格层）

本文件定义 `playbook-and-action-loop` 的成本目标、预算阈值与告警策略。禁止写实现细节；仅定义 MUST/SHALL 与可判定口径。

## 1) 成本维度（最小）

- **存储**：playbook 定义、执行/审批/回滚记录、审计记录与导出包
- **计算**：执行编排、审批校验与漂移检测
- **可观测**：结构化日志与审计导出带来的 IO/存储成本

## 2) 预算阈值（最小）

以下阈值用于早期商业化基线（Gate-2）；后续可按真实负载修订，但口径必须可执行、可判定：

- 控制面请求量：日均 `ls_playbook_action_requests_total` SHOULD ≤ 5,000,000
- 执行量：日均 `ls_playbook_executions_total` SHOULD ≤ 1,000,000
- 单次审计导出体量：单次导出 SHOULD ≤ 1GB（超过则必须分片/分页导出；策略实现可选）
- 单条执行/审批记录体量：序列化后的 `ExecutionRecord`/`ApprovalRecord` SHOULD ≤ 64KB（超过则必须拒绝或降级）

## 3) 告警策略（最小）

- 超过预算阈值（连续 3 个评估窗口）MUST 告警，并形成可审计事件。
- 告警 MUST 可关联到：时间窗口、影响面（执行/审批/回滚/导出）、拒绝原因/阻断类型（若适用）。
