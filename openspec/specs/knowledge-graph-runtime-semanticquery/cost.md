# 知识图谱运行时语义查询 — 成本目标与预算阈值（规格层）

本文件定义 `knowledge-graph-runtime-semanticquery` 的成本目标、预算阈值与告警策略。禁止写实现细节；仅定义 MUST/SHALL 与可判定口径。

## 1) 成本维度（最小）

- **存储**：审计记录与导出包、一致性声明与延迟 SLO 定义
- **计算**：图谱检索、排序/聚合与漂移检测
- **可观测**：结构化日志与审计导出带来的 IO/存储成本

## 2) 预算阈值（最小）

以下阈值用于早期商业化基线（Gate-2）；后续可按真实负载修订，但口径必须可执行、可判定：

- 查询请求量：日均 `ls_kg_semanticquery_requests_total` SHOULD ≤ 20,000,000
- 漂移事件量：日均 `ls_kg_semanticquery_drift_events_total` SHOULD ≤ 10,000
- 单次审计导出体量：单次导出 SHOULD ≤ 1GB（超过则必须分片/分页导出；策略实现可选）
- 单次查询响应体量：序列化后的 `SemanticQueryResponse` SHOULD ≤ 256KB（超过则必须拒绝或降级）

## 3) 告警策略（最小）

- 超过预算阈值（连续 3 个评估窗口）MUST 告警，并形成可审计事件。
- 告警 MUST 可关联到：时间窗口、影响面（查询/漂移/导出）、拒绝原因/阻断类型（若适用）。
