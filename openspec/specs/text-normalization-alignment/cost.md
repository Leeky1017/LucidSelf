# 文本归一化与对齐 — 成本目标与预算阈值（规格层）

本文件定义 `text-normalization-alignment` 的成本目标、预算阈值与告警策略。禁止写实现细节；仅定义 MUST/SHALL 与可判定口径。

## 1) 成本维度（最小）

- **存储**：规则集、对齐映射、变换审计记录、覆盖率报告与例外清单、审计导出包
- **计算**：归一化变换、对齐映射、冲突检测与覆盖率统计
- **可观测**：日志与审计导出带来的 IO/存储成本

## 2) 预算阈值（最小）

以下阈值用于资产门禁的早期商业化基线；后续可按真实负载修订，但口径必须可执行、可判定：

- 变换请求量：日均 `ls_text_norm_requests_total` SHOULD ≤ 5,000,000
- 审计记录写入量：日均 `ls_text_norm_audit_records_total` SHOULD ≤ 1,000,000
- 审计导出体量：单次导出 SHOULD ≤ 1GB（超过则必须分片/分页导出；策略实现可选）

## 3) 告警策略（最小）

- 超过预算阈值（连续 3 个评估窗口）MUST 告警，并形成可审计事件。
- 告警 MUST 可关联到：时间窗口、影响面（变换/对齐/导出）、失败/阻断类型（若适用）。
