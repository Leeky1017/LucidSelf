# 身份与数据隔离 — 成本目标与预算阈值（规格层）

本文件定义 `identity-and-data-isolation` 的成本目标、预算阈值与告警策略。禁止写实现细节；仅定义 MUST/SHALL 与可判定口径。

## 1) 成本维度（最小）

- **存储**：决策记录、违规事件、删除/保留审计、审计导出
- **计算**：鉴权决策与隔离校验
- **可观测**：日志与审计导出带来的 IO/存储成本

## 2) 预算阈值（最小）

以下阈值用于早期商业化基线（Gate-0）；后续可按真实负载修订，但口径必须可执行、可判定：

- 审计记录写入量：日均 `ls_identity_isolation_audit_records_total` SHOULD ≤ 10,000,000
- 隔离违规事件：日均 `ls_identity_isolation_violations_total` SHOULD ≤ 1,000（异常抬升必须告警）
- 审计导出体量：单次导出 SHOULD ≤ 5GB（超过则必须分片/分页导出；策略实现可选）

## 3) 告警策略（最小）

- 超过预算阈值（连续 3 个评估窗口）MUST 告警，并形成可审计事件。
- 告警 MUST 可关联到：时间窗口、影响面（鉴权/审计/导出）、关键标识符（若适用，至少 `org_id` 与时间窗口）。
