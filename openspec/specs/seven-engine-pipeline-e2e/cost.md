# 七引擎流水线E2E — 成本目标与预算阈值（规格层）

本文件定义 `seven-engine-pipeline-e2e` 的成本目标、预算阈值与告警策略。禁止写实现细节；仅定义 MUST/SHALL 与可判定口径。

## 1) 成本维度（最小）

- **存储**：阶段契约、交接校验记录、trace 元数据、失败恢复记录、审计日志与导出包
- **计算**：交接校验、回放/恢复校验、门禁检查与偏差检测
- **可观测**：结构化日志与 trace/span 带来的 IO/存储成本

## 2) 预算阈值（最小）

以下阈值用于 Gate-2 的最小商业化基线；后续可按真实负载修订，但口径必须可执行、可判定：

- 单次 E2E 执行审计体量：单次 `run_id` 关联的审计记录导出 SHOULD ≤ 50MB（超过则必须分片/分页导出；策略实现可选）
- 单条交接校验记录体量：序列化后的 `HandoffValidationRecord` SHOULD ≤ 64KB（超过则必须拒绝或降级）
- 单条 trace 元数据体量：序列化后的 `E2ETraceLink` SHOULD ≤ 256KB（超过则必须拒绝或降级）

## 3) 告警策略（最小）

- 超过预算阈值（连续 3 个评估窗口）MUST 告警，并形成可审计事件。
- 告警 MUST 可关联到：时间窗口、影响面（存储/计算/导出）、拒绝原因/阻断类型（若适用）。
