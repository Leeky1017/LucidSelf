# 跨书一致性与冲突 — 成本目标与预算阈值（规格层）

本文件定义 `cross-book-consistency-conflicts` 的成本目标、预算阈值与告警策略。禁止写实现细节；仅定义 MUST/SHALL 与可判定口径。

## 1) 成本维度（最小）

- **存储**：冲突记录、来源引用、决策与例外批准记录、冲突报表、审计记录与导出包
- **计算**：冲突检测、报表聚合、审计导出
- **可观测**：日志与审计导出带来的 IO/存储成本

## 2) 预算阈值（最小）

以下阈值用于资产门禁的早期商业化基线；后续可按真实负载修订，但口径必须可执行、可判定：

- 冲突发现量：日均 `ls_cross_book_conflicts_total` SHOULD ≤ 5,000,000
- 例外批准量：日均 `ls_cross_book_exception_approvals_total` SHOULD ≤ 100,000
- 审计导出体量：单次导出 SHOULD ≤ 1GB（超过则必须分片/分页导出；策略实现可选）

## 3) 告警策略（最小）

- 超过预算阈值（连续 3 个评估窗口）MUST 告警，并形成可审计事件。
- 告警 MUST 可关联到：时间窗口、影响面（检测/决策/导出）、失败/阻断类型（若适用）。
