# 引用与证据协议 — 成本目标与预算阈值（规格层）

本文件定义 `citations-evidence-protocol` 的成本目标、预算阈值与告警策略。禁止写实现细节；仅定义 MUST/SHALL 与可判定口径。

## 1) 成本维度（最小）

- **存储**：证据链、溯源记录、审计记录与导出包
- **计算**：引用锚点解析、证据链校验与偏差检测
- **可观测**：日志与审计导出带来的 IO/存储成本

## 2) 预算阈值（最小）

以下阈值用于早期商业化基线（Gate-1）；后续可按真实负载修订，但口径必须可执行、可判定：

- 证据链写入量：日均 `ls_citations_evidence_requests_total{event_type="EVIDENCE_CHAIN_CREATE"}` SHOULD ≤ 5,000,000
- 审计导出体量：单次导出 SHOULD ≤ 1GB（超过则必须分片/分页导出；策略实现可选）
- 单条证据链体量：序列化后的 `EvidenceChain`（含引用锚点集合）SHOULD ≤ 256KB（超过则必须拒绝或降级）

## 3) 告警策略（最小）

- 超过预算阈值（连续 3 个评估窗口）MUST 告警，并形成可审计事件。
- 告警 MUST 可关联到：时间窗口、影响面（校验/生成/导出）、拒绝原因/阻断类型（若适用）。
