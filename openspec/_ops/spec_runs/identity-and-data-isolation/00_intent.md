# identity-and-data-isolation — Intent Alignment

## 目的（可验收语言复述）

`identity-and-data-isolation` 规格用于把 LS 的 `user_id`/`org_id` 身份范围与多租户隔离固化为可审计的契约与门禁，确保任何数据读写与用户可见输出都不会跨越授权边界，并且可追责、可回放、可导出审计证据。

## 边界（不做什么）

- 本 spec 仅定义 MUST/SHALL 的契约、字段语义、门禁与验收口径。
- 本 spec 不规定具体实现细节、代码路径、技术选型或伪代码。

## 成功条件（可判定）

- 关键产物（身份声明、访问控制决策记录、隔离违规审计、删除/保留审计）具备字段级约束与完整性语义。
- 对外契约字段与拒绝原因以机器可读形式固定，并可通过命令验证。
- 可观测/审计字段与脱敏规则明确，且可按 `user_id`/`org_id`/`version_id`（及 `request_id/trace_id` 如适用）关联定位。
- Gate-0 相关验收口径可执行；失败具备明确阻断策略。
