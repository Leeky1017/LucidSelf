# security-privacy-compliance — Intent Alignment

## 目的（可验收语言复述）

`security-privacy-compliance` 规格用于把 LS 的安全、隐私与合规要求固化为可审计的契约与门禁，确保数据最小化、访问控制、隐私请求（删除/导出等）与事件响应具备可追溯证据链，并对越权/违规形成确定性的阻断与追责。

## 边界（不做什么）

- 本 spec 仅定义 MUST/SHALL 的契约、字段语义、门禁与验收口径。
- 本 spec 不规定具体实现细节、代码路径、技术选型或伪代码。

## 成功条件（可判定）

- 关键产物（数据分类标签、访问控制证据、隐私请求处理记录、事件响应记录）具备字段级约束与完整性语义。
- 对外契约字段与拒绝原因以机器可读形式固定，并可通过命令验证。
- 可观测/审计字段与脱敏规则明确，且可按 `user_id`/`org_id`/`version_id`（及 `request_id/trace_id` 如适用）关联定位。
- Gate-0 相关验收口径可执行；失败具备明确阻断策略。
