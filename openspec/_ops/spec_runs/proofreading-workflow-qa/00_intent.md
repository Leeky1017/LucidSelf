# proofreading-workflow-qa — Intent Alignment

## 目的（可验收语言复述）

`proofreading-workflow-qa` 规格用于把 LS 的「语料与抽取结果的校对/复核/批准」固化为可审计的资产流程：定义校对任务、缺陷记录、批准记录与 QA 指标报表的最小契约与字段约束，并确保所有质量处置可追溯到 `corpus_release_id/version_id`（以及证据链/来源），从而为 `资产门禁` 的“可追溯资产质量 + 可审计批准链 + 可导出最小证据包”提供可执行、可阻断、可复现的验收口径。

## 边界（不做什么）

- 本 spec 仅定义 MUST/SHALL 的契约、字段语义、门禁与验收口径。
- 本 spec 不规定具体实现细节、代码路径、技术选型或伪代码。

## 成功条件（可判定）

- 关键产物（校对任务、缺陷记录、批准记录、QA 指标报表）具备字段级约束与完整性语义。
- 对外契约字段与拒绝原因以机器可读形式固定，并可通过命令验证。
- 资产门禁验收口径可执行；失败具备明确阻断策略并可追溯到 `corpus_release_id/version_id`。
