# production-defaults-no-mock — Intent Alignment

## 目的（可验收语言复述）

`production-defaults-no-mock` 规格用于把 LS 的生产默认值与环境隔离要求固化为可审计契约：生产环境禁止默认 mock/静默降级，配置必须可校验且失败可解释，并对违规行为形成阻断与审计记录，确保 Gate-0 的“可运行且可信默认值”基线。

## 边界（不做什么）

- 本 spec 仅定义 MUST/SHALL 的契约、字段语义、门禁与验收口径。
- 本 spec 不规定具体实现细节、代码路径、技术选型或伪代码。

## 成功条件（可判定）

- 关键产物（配置校验规则、环境隔离约束、启动失败原因记录、违规审计）具备字段级约束与完整性语义。
- 对外契约字段与拒绝原因以机器可读形式固定，并可通过命令验证。
- 失败处理策略明确：生产默认值违规 MUST 阻断启动/晋级，并留下可行动证据。
