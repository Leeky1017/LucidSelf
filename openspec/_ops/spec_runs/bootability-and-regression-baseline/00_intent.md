# bootability-and-regression-baseline — Intent Alignment

## 目的（可验收语言复述）

`bootability-and-regression-baseline` 规格用于把 LS 的“可启动性”与“最小回归基线”固化为可执行门禁：明确启动检查清单、回归套件基线、失败阻断记录与版本关联报告的字段语义与验收口径，使 Gate-1 的回归能力可重复执行、可审计、可阻断。

## 边界（不做什么）

- 本 spec 仅定义 MUST/SHALL 的契约、字段语义、门禁与验收口径。
- 本 spec 不规定具体实现细节、代码路径、技术选型或伪代码。

## 成功条件（可判定）

- 关键产物（启动检查清单、回归套件基线、失败阻断记录、版本关联报告）具备字段级约束与完整性语义。
- 对外契约字段与拒绝原因以机器可读形式固定，并可通过命令验证。
- Gate-1 的验收口径可执行；失败具备明确阻断策略并可追溯到版本。
