# engine-id-governance — Intent Alignment

## 目的（可验收语言复述）

`engine-id-governance` 规格用于把 LS 系统中 `engine_id` 的**注册、唯一性与生命周期治理**固化为可审计的契约与门禁，使任何引擎相关的输出/作业都能被稳定追溯与治理。

## 边界（不做什么）

- 本 spec 仅定义 MUST/SHALL 的契约、字段语义、门禁与验收口径。
- 本 spec 不规定具体实现细节、代码路径、技术选型或伪代码。

## 成功条件（可判定）

- 关键产物（注册表、生命周期变更、弃用/替代关系、审计记录）在规格层具备字段级约束与完整性语义。
- 对外契约字段与拒绝原因以机器可读形式固定，并可通过命令验证。
- 可观测/审计字段与脱敏规则明确，且可按 `engine_id`/`version_id`（及 `user_id` 如适用）关联定位。
- Gate-0 相关验收口径可执行；失败具备明确阻断策略。
