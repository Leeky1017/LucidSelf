# 项目愿景

> Life Version Tree: 让人生选择可视化、可追溯

---

## 核心问题

**传统命理的困境：**
- 给你一个"定论"，你只能信或不信
- 预测准了是"命中注定"，不准是"时候未到"
- 用户被动接受，没有参与感

**用户真正想要的：**
- 我有哪些可能性？
- 每条路的代价和收益是什么？
- 我过去的选择如何影响了现在？
- 我能不能"改命"？

---

## LV 的回答

### 不是预测，是"可能性地图"

传统命理说：你财运不好  
LV 说：你的财富路径有 3 种可能——

1. **稳健路线**：利用「正财」特质做长期积累，适合工资+理财
2. **冒险路线**：你有「偏财」潜力，但「比劫」会带来合伙风险
3. **转化路线**：通过「食伤生财」走技术变现

每条路都有因子支撑，每条路都有风险提示。

### 不是一次性报告，是"版本演化"

用户的每次选择、每次行为反馈，都会被记录：
- 选了"冒险路线"但投资失败 → 不是推翻预测，而是分析"哪个因素在起作用"
- 可能是流年「七杀」冲克，可能是「比劫夺财」的合伙问题

---

## 商业价值

### 差异化：

| 竞品 | LV |
|------|-----|
| 一次性报告 | 持续版本管理 |
| 静态预测 | 动态推演 |
| "你会怎样" | "你可以怎样" |
| 用户被动 | 用户参与决策 |

### 目标用户：

- 不迷信但对自我认知感兴趣
- "结构控"、喜欢分析和复盘
- 愿意为长期个人管理付费（类似健身 App、日记 App）

---

## 与 LS 完整系统的关系

LV 是 LS 的**核心主干**：

```
LS 完整系统
├── Playbook（命理解读报告）← 可由 LV 因子生成
├── Dream Journal（梦境日记）← 为 LV 提供校准数据
├── Life Version Tree ← ✅ MVP 核心
└── 典籍精校 ← 为 LLM 叙事提供素材（后续）
```

MVP 先做 LV，验证核心交互后再扩展其他模块。
