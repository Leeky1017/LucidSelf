# 中文典籍深度精校模板 V3.0

> **版本**: 3.0 | **创建日期**: 2026-01-27
> 
> **设计原则**: 从"填表式机械精校"升级为"深度理解+批判分析+跨体系论证+案例验证"的学术级精校框架

---

## 使用约定（必须遵守）

### 核心变化（相对于 V2.0）

| 维度 | V2.0 | V3.0 |
|:---|:---|:---|
| 粒度 | 一句一卡 | 一个语义单元一卡 |
| 结构 | A-F 六段填表 | L1-L6 六层深度分析 |
| 因子提取 | 直接从字面提取 | 先理解后提取，需论证 |
| 跨体系映射 | 无需依据 | 必须提供学术论证 |
| 验证 | 无 | 至少1个案例验证 |

### 语义单元定义

**语义单元** = 一个完整的、可独立理解的论述，包含：
- 一个核心论点/命题
- 支撑该论点的论证/举例
- （可选）该论点的边界/例外说明

**划分原则**：
1. **完整性**：包含论点+论证+（边界）
2. **独立性**：单独阅读可理解
3. **原子性**：再拆分会破坏论证结构
4. **可引用性**：可被其他地方独立引用

### 深度级别

| 级别 | 适用对象 | 模板要求 |
|:---|:---|:---|
| **Scholarly** | 核心概念（P0）| 完整 L1-L6 所有字段 |
| **Deep** | 主干概念（P1）| L1+L3+L4+L5 简化版 |
| **Standard** | 补充概念（P2）| L1+L3 核心字段 |
| **Index** | 索引条目（P3）| 仅 S 综述 |

### 产出位置

- 精校卡统一放在 `典籍/calibrated/cards/<book_abbr>/`
- 文件命名：`<book_abbr>_<concept>_<seq>.md`

### 禁止项（硬性）

- 禁止将过渡句、引导句单独成卡
- 禁止因子提取无论证
- 禁止跨体系映射无依据
- 禁止规则草案不可操作
- 禁止忽略适用边界和局限性

## L3 - 语义本质提炼层 (Semantic Essence)

### L3.1 核心概念解构

| 概念 | 字面意义 | 深层语义 | 现代重构 |
|:---|:---|:---|:---|
| `<术语>` | <字典/训诂定义> | <在此语境中的真正含义，作者赋予的特殊意涵> | <用现代语言/概念重述，可理解、可操作> |

### L3.2 抽象规则提取

**规则ID**: `rule_<book_abbr>_<semantic_tag>`

**规则结构**（伪代码形式）：

```
IF <触发条件：使用因子表达式，明确具体>
AND <情境条件：什么场景/语境下适用>
THEN <判断结论：具体、可操作的结论>
WITH confidence = <High/Medium/Low>
BECAUSE <证据引用：原文出处>
UNLESS <例外条件：什么情况下规则不适用>
```

**规则操作化说明**：

| 维度 | 说明 |
|:---|:---|
| 触发条件操作化 | <如何在实际中判断条件是否满足？需要什么输入？> |
| 结论操作化 | <结论意味着什么具体判断或建议？如何呈现给用户？> |
| 置信度理由 | <为什么是这个置信度？依据是什么？> |

## 质量检查清单（精校完成后自检）

### Scholarly 级别检查清单

- [ ] **S 综述**：核心论点一句话准确概括了语义单元的洞见
- [ ] **L1.1**：原文完整，上下文分析清晰
- [ ] **L1.2**：历史背景和学术脉络有据可查
- [ ] **L1.3**：作者意图和修辞策略分析合理
- [ ] **L2.1**：论证逻辑评估诚实，不回避存疑之处
- [ ] **L2.2**：与至少1本其他典籍进行了对比分析
- [ ] **L2.3**：明确了适用边界和局限性
- [ ] **L3.1**：核心概念的深层语义超越了字面意思
- [ ] **L3.2**：规则可操作，能写出伪代码
- [ ] **L3.3**：因子提取有论证，不是简单拆词
- [ ] **L4.1**：跨体系映射基于功能等价，有学术依据
- [ ] **L4.3**：中立标签映射有逻辑论证
- [ ] **L5.1**：至少1个案例验证
- [ ] **L5.3**：引用了相关学术文献或经典著作
- [ ] **L6.2**：诚实标注了各层次的不确定性
- [ ] **L6.3**：列出了待进一步研究的问题
